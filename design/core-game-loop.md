## 设计文档

**版本**: 1.0  
**设计师**: 创意总监  
**日期**: 2026-04-22

---

## 1. 概述

### 1.1 目标
设计一个让玩家体验 AI 创业全过程的模拟经营游戏，核心是"雇佣 Agent→接项目→交付赚钱"的循环。

### 1.2 背景
AI 热潮下，很多人对 AI 创业感兴趣但缺乏了解。游戏通过模拟方式，让玩家体验：
- 如何管理 AI 团队
- 如何平衡项目质量/速度/成本
- 如何应对创业挑战

### 1.3 范围
- **包含**: 核心循环、Agent 系统、项目系统、事件系统
- **不包含**: 多人联机、复杂 3D 画面、长篇剧情

---

## 2. 核心循环

### 2.1 短期循环 (每次操作)
```
接项目 → 分配 Agent → 开发 → 交付 → 收款 → 反馈
   ↓                                        ↑
   └────────────────────────────────────────┘
```

### 2.2 中期循环 (每天)
```
早上：查看财务状况 → 雇佣/培训 → 接新项目
中午：监控项目进度 → 处理事件
晚上：结算收益 → 保存进度
```

### 2.3 长期循环 (整个游戏)
```
车库创业 → 小办公室 → 科技公司 → AI 帝国
  ↓           ↓           ↓          ↓
5 Agent     15 Agent    30 Agent    50+ Agent
```

## 3. 核心机制

### 3.1 Agent 系统

#### Agent 属性
| 属性 | 说明 | 影响 |
|------|------|------|
| **效率** | 工作效率 | 开发速度 |
| **创造力** | 创新能力 | 项目质量 |
| **稳定性** | 工作稳定性 | 出错概率 |
| **满意度** | 工作满意度 | 离职概率 |

#### Agent 类型
| 类型 | 效率 | 创造力 | 薪资/天 |
|------|------|--------|---------|
| 初级 Agent | ⭐⭐ | ⭐⭐ | $1k |
| 中级 Agent | ⭐⭐⭐ | ⭐⭐⭐ | $2k |
| 高级 Agent | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | $4k |
| 专家 Agent | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | $8k |

### 3.2 项目系统

#### 项目类型
| 类型 | 周期 | 报酬 | 难度 |
|------|------|------|------|
| 网站开发 | 3-7 天 | $5-20k | ⭐⭐ |
| 移动 App | 2-4 周 | $20-50k | ⭐⭐⭐ |
| AI 系统集成 | 1-2 周 | $10-30k | ⭐⭐⭐⭐ |
| 企业软件 | 1-3 月 | $50-200k | ⭐⭐⭐⭐⭐ |

#### 项目难度影响
```
难度 ⭐: 需要 1 个 Agent，效率 100%
难度 ⭐⭐: 需要 2 个 Agent，效率 90%
难度 ⭐⭐⭐: 需要 3 个 Agent，效率 80%
难度 ⭐⭐⭐⭐: 需要 4 个 Agent，效率 70%
难度 ⭐⭐⭐⭐⭐: 需要 5 个 Agent，效率 60%
```

### 3.3 财务系统

#### 收入来源
- 项目交付 (主要)
- 客户小费 (随机)
- 投资融资 (后期)

#### 支出项目
- Agent 薪资 (每日)
- 办公室租金 (每周)
- 设备采购 (一次性)
- 研发投入 (一次性)

#### 现金流管理
```
现金 > 0: 正常运营
现金 < 0: 贷款利息
现金 < -50k: 破产风险
```

## 4. 事件系统

### 4.1 正面事件 (概率 30%)

| 事件 | 效果 | 概率 |
|------|------|------|
| 客户给小费 | +10% 报酬 | 10% |
| Agent 灵感爆发 | 效率 x2 (1 天) | 5% |
| 技术突破 | 解锁新能力 | 5% |
| viral 传播 | 新客户 +50% | 5% |
| 政府补贴 | +$10k | 5% |

### 4.2 负面事件 (概率 20%)

| 事件 | 效果 | 概率 |
|------|------|------|
| 需求变更 | 工期 +50% | 8% |
| Bug 爆发 | 返工 (1 天) | 5% |
| Agent 生病 | 请假 3 天 | 3% |
| 服务器宕机 | 进度 -20% | 2% |
| 客户差评 | 口碑 -20% | 2% |

## 6. 胜利条件

### 6.1 主要胜利
```
达到以下任一条件:
- 公司估值 $100M+
- 拥有 50+ Agent
- 完成 100 个项目
- 连续 30 天盈利
```

### 6.2 失败条件
```
触发以下任一条件:
- 现金 < -$50k (破产)
- 口碑 < 2.0 (倒闭)
- 连续 7 天无项目 (资金链断裂)
```

## 8. 边界情况

### 8.1 极端情况处理
- **所有 Agent 离职**: 游戏结束提示
- **连续失败**: 触发新手保护 (简单项目)
- **刷钱漏洞**: 每日项目数量上限

### 8.2 存档机制
- 自动保存 (每日结束)
- 手动保存 (随时)
- 存档槽位：3 个
